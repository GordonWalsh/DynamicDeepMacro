# Project Handover: Macro Engine Compiler
As of 2026-04-08 10:30

## 1\. Project Goal

Development of a deterministic, context-aware, text-replacement compiler (the "Macro Engine") designed to handle highly nested, randomized structural logic for Generative AI prompting (specifically integrating as a Python wrapper for ComfyUI). The engine resolves custom syntax for dynamic string building, scoped variable definitions, and path-hashed Pseudo-Random Number Generation (PRNG) branch selection using a strict Breadth-Eager, Depth-Lazy architecture.

## 2\. Current Technical Stack

*   **Language:** Python 3
*   **Key Native Libraries:** `re` (for unbounded pattern replacements), `random` (for PRNG), `collections.deque` (for LIFO Context Stack), `codecs` (for unicode escape sequence decoding).
*   **Target Environment:** ComfyUI custom node ecosystem / Python microservice.

## 4\. Key Decisions & Realizations (The 'Why' and 'Gotchas')

*   **Breadth-Eager, Depth-Lazy Execution:** The Lexer and Parser must _only_ operate at the zero-depth of the current string. The internal contents of an invocation (`< >`) or group (`{ }`) are treated as inert strings until explicitly evaluated. This prevents the compiler from wasting CPU cycles building ASTs for PRNG branches that are not selected.
*   **Interval-Tracking Speculative Lexer:** Buffering characters in Python is slow. The Lexer must track start/end indices in a single pass, utilizing an "Interval Culling" algorithm where any boundary marker falling strictly inside a higher-order boundary is ignored. This natively protects nested syntax and isolates user typos (unclosed brackets).
*   **Scope Hoisting (Footnote Architecture):** The Parser decouples State from Data. Definitions (`:key:val`) found at zero-depth are stripped from the output list and attached to the parent node's `local_definitions`. They are pushed to the Context Stack exactly when the node evaluates, allowing users to place definitions anywhere in the block (e.g., at the bottom) without leaking state early.
*   **Dual-Accumulator Context Stack:** To support dynamic array building without recursive stack-overflow traps, the stack searches Head-to-Tail (Strongest to Weakest). It accumulates Left (`<:`) and Right (`>:`) concatenations until it hits a Base terminator (`:` or `::`).
*   **Inside-Out Concatenation:** By evaluating the Context Stack's returned list sequentially (from Strong to Weak), the strongest modifiers naturally bind tightest to the base string, preserving lexical scoping without the Context Stack needing to manage string buffers.
*   **The Regex Identity Trap:** Unbounded Pre/Post patterns (`:<`, `:>`) cannot accumulate each other like bounded macros, as they are search rules, not explicit keys. Applying `<:` or `>:` (`<::` or `>::` in Weak form) to them acts as a syntactic shorthand to automatically inject the `\g<0>` capture token, preserving the matched text safely.
*   **Path-Hashed PRNG Determinism:** A global random counter breaks determinism when branches are added or removed. The PRNG seed for any child evaluation must be computed dynamically: `Hash(parent_seed + child_index)`.
*   **Sandboxed Regex Definitions (`/ /`):** To avoid the "Slash Collision Trap" (accidentally parsing file paths or URLs in general text as regex), the `/ /` syntax is completely inert in standard literals. Inside Definition keys and values, wrapping text exactly in `/ /` compiles it as Regex. If a literal definition needs to start with a slash, the user escapes the leading character (`\/`), which bypasses the regex compiler and treats the entire remaining string as a pure literal.
*   **Late-Binding Anonymous Escape Block (`</.../>`):** To allow dynamic injection of formatting (like `\n` or `\t`) without risking global parsing collisions, the engine uses a late-binding sandbox. If an Invocation node's key is exactly wrapped in slashes (eg `</\n/>` for `\n`), it bypasses the Context Stack lookup, strips the outer slashes, and runs the inner payload through Python's native `codecs.decode(..., 'unicode_escape')` to safely resolve the literal byte sequence into the final string.
*   **The Grandchild Trap & Base Template Pattern:** To avoid creating unnecessary dummy wrappers (`BlockNode`) at every evaluation step, the Parser does not return a single root node for its parsed sub-trees. It simply returns `Tuple[list[Definition], list[ASTNode]]`. The boilerplate scope lifecycle (push definitions -> iterate children -> pop definitions) is abstracted into a protected base class method (`ASTNode._evaluate_scope`). This keeps the AST perfectly flat, prevents duplicated `for`\-loops across different node types, and reduces memory overhead.
